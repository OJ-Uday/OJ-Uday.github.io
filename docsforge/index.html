docsforge

Reusable GitHub Actions workflow. Discovers .docmeta.yml files in a caller repo, dispatches each doc to the matching builder (LaTeX / node-docx / markdown-pandoc), runs the whole matrix in parallel, and publishes outputs to a rolling latest release. Consumer wiring is three lines of YAML.

Design choices

  • Fail-open discovery — malformed .docmeta.yml warns; other docs keep building.
  • Lockfile-survivor mode — node-docx auto-heals package-lock.json files that point at private registries the runner can't reach.
  • Parallel matrix — N docs → N concurrent legs, fail-fast off.
  • Concurrency-cancel — newer pushes preempt in-flight runs.

Consumer wiring

jobs:
  build:
    uses: OJ-Uday/docsforge/.github/workflows/build-docs.yml@main

Full architecture, .docmeta.yml schema, and a Mermaid diagram live in the docsforge README.
