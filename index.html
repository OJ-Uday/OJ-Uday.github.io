01 Systems I've shipped

vetlock — supply-chain behavioral diff

Reads two lockfiles, fetches every changed package, safely extracts each tarball, and reports capability deltas: new install scripts, new network endpoints, env-token harvest, obfuscation jumps, integrity tampering. 12 of 13 real npm supply-chain attacks caught by the replay corpus. 39 of 48 red-team exploits closed across a six-agent adversarial review. Ships with a NEVER-EXECUTE canary — the analyzer never runs code from analyzed packages, tested every build.

Operational-intelligence platform

Browser SDK → Go + ClickHouse backend → React dashboard, across a federated micro-frontend estate of 5+ applications. Two-tier element identity and capture-time causality feed an inference engine that names the UI automatically — analysts approve with one click, and a new app onboards with two HTML attributes, zero code changes.

05 Writing

  • The vetlock red-team review

    Six adversarial agents, 48 exploit candidates, an adversarial verifier that defaulted to REFUTED — 35 confirmed end-to-end. 39 of 48 catalogued exploits closed as of v0.4.2. What the numbers said the tool wasn't yet.

  • When correctness beats performance

    A 3x-faster string-distance library looked like an easy win — until "attackers rarely transpose" collided with npm's own typosquat history. Reverted, with tests that pin every transposition case in place — the principle that governs every security-boundary choice in vetlock.

  • Self-building instrumentation

    Hand-written analytics loses to features. Auto-capture breaks on every redesign. Nothing captures what a click was for. A sense-understand-act loop that names the UI from its own witnessed effects — two-tier identity + capture-time causality make that safe.

  • The two-identity problem in zero-touch delivery

    Two GitHub rules — no self-approval, and GITHUB_TOKEN events don't cascade — quietly break naive deploy automation: the PR opens, required checks never fire, auto-merge stalls forever. Four arrangements that survive them.
